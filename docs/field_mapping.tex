\section{Prérequis}

\begin{enumerate}[leftmargin=2em, itemsep=4pt]
  \item Un compte Jira avec accès en lecture au projet \textbf{IRPAUTO}.
  \item Un \textbf{API Token} Jira (généré sur \texttt{id.atlassian.com} $\to$ Security $\to$ API tokens).
  \item L'URL de l'instance Jira : \texttt{https://dxc-insurance-delivery.atlassian.net}
  \item Python 3.x avec les packages \texttt{requests} et \texttt{pandas}.
\end{enumerate}

\section{Étapes de Connexion}

\subsection{1. Authentification}

L'API Jira REST v3 utilise une authentification \textbf{Basic Auth} avec l'email du compte
et le token API (pas le mot de passe).

\begin{purplebox}
\small\ttfamily
auth = HTTPBasicAuth("prenom.nom@dxc.com", "VOTRE\_API\_TOKEN")\\
headers = \{"Accept": "application/json", "Content-Type": "application/json"\}
\end{purplebox}

\subsection{2. Endpoint de recherche}

Utiliser l'endpoint \textbf{POST} (la version GET est dépréciée pour de grands volumes) :

\begin{purplebox}
\small\ttfamily
POST \{jira\_url\}/rest/api/3/search/jql
\end{purplebox}

\subsection{3. Corps de la requête}

\begin{purplebox}
\small\ttfamily
\{\\
\quad "jql": "project = IRPAUTO ORDER BY updated ASC",\\
\quad "maxResults": 100,\\
\quad "fields": [liste des field IDs ci-dessous]\\
\}
\end{purplebox}

\subsection{4. Pagination}

La réponse contient un champ \texttt{nextPageToken}. Tant qu'il est présent et que
\texttt{isLast} est \texttt{false}, relancer la requête en ajoutant
\texttt{"nextPageToken": valeur} au corps. Le projet IRPAUTO contient plus de
55\,000 tickets — la pagination est obligatoire.

\subsection{5. Synchronisation incrémentale}

Pour ne récupérer que les tickets modifiés depuis la dernière synchronisation,
ajouter un filtre JQL :

\begin{purplebox}
\small\ttfamily
"jql": "project = IRPAUTO AND updated >= \textbackslash"2026-04-01\textbackslash" ORDER BY updated ASC"
\end{purplebox}

\subsection{6. Trouver l'ID d'un champ personnalisé}

Pour identifier l'ID d'un champ custom (\texttt{customfield\_XXXXX}) :
\begin{enumerate}[leftmargin=2em, itemsep=2pt]
  \item Ouvrir n'importe quel ticket dans Jira
  \item Ajouter \texttt{.json} à la fin de l'URL du ticket dans le navigateur
  \item Dans le JSON, chercher le nom du champ dans la section \texttt{names} ou \texttt{fields}
  \item Ou appeler : \texttt{GET \{jira\_url\}/rest/api/3/field} pour la liste complète
\end{enumerate}

\section{Notes Importantes}

\begin{enumerate}[leftmargin=2em, itemsep=6pt]

  \item \textbf{Story Points — double champ :}
        Pour les tickets IRPAUTO, \texttt{customfield\_10049} (numérique) est vide
        sur environ 82\% des tickets. Les vraies valeurs se trouvent dans
        \texttt{customfield\_10520} (liste déroulante). Il faut lire les deux et
        fusionner : prendre \texttt{10049} en priorité, et remplir les NULL avec
        la valeur numérique extraite de \texttt{10520}.

  \item \textbf{Champs ADF (rich-text) :}
        Les champs de type description (Root Cause Description, Impact Description,
        etc.) sont retournés au format \textbf{Atlassian Document Format (ADF)},
        une structure JSON imbriquée. Il faut les parcourir récursivement pour
        en extraire le texte brut.

  \item \textbf{Champs SLA :}
        L'API retourne des objets SLA complexes contenant des cycles de temps, pas
        les chaînes formatées ``Xh Ym'' visibles dans l'export Excel. Stocker en
        NULL ou implémenter un parseur dédié si nécessaire.

  \item \textbf{Dates :}
        Toutes les dates Jira sont retournées en ISO 8601 avec offset timezone
        (ex. \texttt{2026-03-15T14:30:00+0200}). Les convertir en datetime
        timezone-aware puis supprimer la timezone pour la base de données :
        \texttt{pd.to\_datetime(..., utc=True).dt.tz\_convert(None)}.

  \item \textbf{Champs objet :}
        La plupart des champs personnalisés retournent un dictionnaire JSON.
        Extraire selon le type : \texttt{name} pour les listes, \texttt{displayName}
        pour les utilisateurs, \texttt{value} pour les listes déroulantes.

\end{enumerate}
